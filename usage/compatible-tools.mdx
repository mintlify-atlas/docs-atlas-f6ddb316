Pico FIDO is designed to be broadly compatible with the existing FIDO2 and Yubikey-compatible ecosystem. Because it presents itself as a standard USB HID authenticator, most tools and browsers that support FIDO2 or U2F work without any special configuration. A small set of dedicated tools also gives you full control over credentials, firmware, and device settings.

<Note>
Pico FIDO uses the HID driver, which is built into Windows, macOS, and Linux. You do not need to install any additional drivers to use the device.
</Note>

## Browser support

All major browsers support FIDO2/WebAuthn over USB HID and work with Pico FIDO without any plugins or configuration:

| Browser | FIDO2/WebAuthn support |
|---|---|
| Google Chrome | Yes |
| Mozilla Firefox | Yes |
| Apple Safari | Yes |
| Microsoft Edge | Yes |

## Operating system support

| Platform | Support | Notes |
|---|---|---|
| Windows | Full | HID driver built in; no additional setup needed |
| macOS | Full | HID driver built in; no additional setup needed |
| Linux | Full | HID driver built in; ensure your user has access to the HID device, or add a udev rule |

## OpenSC and CCID compatibility

The default firmware ships with VID/PID `2E8A:10FE` (granted by Raspberry Pi). If you want to use Pico FIDO with OpenSC or other CCID-based tools, you need to add this VID/PID to the `Info.plist` file of your system's CCID driver. The [PicoKey App](https://www.picokeys.com/picokeyapp/) can automate this step.

Alternatively, you can change the VID/PID to one you own at runtime using the `CTAP_CONFIG_PHY_VIDPID` vendor configuration command. See [Vendor configuration and device customization](/usage/vendor-configuration) for details.
