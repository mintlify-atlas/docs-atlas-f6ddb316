title: "FIDO2, CTAP 2.1, and WebAuthn authenticator features"
description: "Pico FIDO implements CTAP 2.1, CTAP 1 (U2F), and WebAuthn on a Raspberry Pi Pico or ESP32, with a full set of CBOR commands, algorithms, and curves."
---

Pico FIDO turns your Raspberry Pi Pico or ESP32 into a fully compliant FIDO2 authenticator. It speaks CTAP 2.1 and CTAP 1 (U2F) over USB HID, satisfies the WebAuthn relying-party requirements, and exposes the complete set of CBOR commands defined in the specification—from credential creation to large-blob writes. Every operation that requires user presence is gated behind a physical button press, and user verification is handled by a PIN with up to eight retries.

## Protocol versions

Pico FIDO advertises the following protocol strings in the `authenticatorGetInfo` response:

- `U2F_V2` — CTAP 1 / U2F backwards compatibility
- `FIDO_2_0` — CTAP 2.0 baseline
- `FIDO_2_1` — CTAP 2.1 with full extension support
- `FIDO_2_2` — CTAP 2.2 draft features

## CBOR commands

The following CTAP CBOR commands are implemented in `ctap.h`:

| Command | Code | Description |
|---|---|---|
| `CTAP_MAKE_CREDENTIAL` | `0x01` | Register a new credential |
| `CTAP_GET_ASSERTION` | `0x02` | Authenticate with an existing credential |
| `CTAP_GET_INFO` | `0x04` | Return authenticator capabilities |
| `CTAP_CLIENT_PIN` | `0x06` | Set, change, or verify a PIN |
| `CTAP_RESET` | `0x07` | Factory-reset the device |
| `CTAP_GET_NEXT_ASSERTION` | `0x08` | Retrieve additional assertions from a previous call |
| `CTAP_CREDENTIAL_MGMT` | `0x0A` | Enumerate and delete resident credentials |
| `CTAP_SELECTION` | `0x0B` | Prompt the user to select this device |
| `CTAP_LARGE_BLOBS` | `0x0C` | Read or write large-blob data |
| `CTAP_CONFIG` | `0x0D` | Configure authenticator-wide settings |

| RS256 | `-257` | RSASSA-PKCS1-v1_5 with SHA-256 |
| RS384 | `-258` | RSASSA-PKCS1-v1_5 with SHA-384 |
| RS512 | `-259` | RSASSA-PKCS1-v1_5 with SHA-512 |

| ESB384 | `-267` | ECDSA with SHA-384 on BP384r1 |

| Curve | ID | Description |
|---|---|---|
| P-256 | `1` | NIST P-256 (secp256r1) |

## PIN UV auth token permissions

PIN UV auth tokens restrict what an authenticated session may do. Pico FIDO enforces the following permission bits from `ctap.h`:

| Permission | Bit | Scope |
|---|---|---|
| `MC` | `0x01` | `authenticatorMakeCredential` |
| `GA` | `0x02` | `authenticatorGetAssertion` |
| `CM` | `0x04` | `authenticatorCredentialManagement` |
| `BE` | `0x08` | Bio enrollment |
| `LBW` | `0x10` | Large-blob write |
| `ACFG` | `0x20` | Authenticator configuration |
| `PCMR` | `0x40` | Per-credential management read-only |

Both PIN protocol 1 and PIN protocol 2 are supported.

## User presence and user verification

Pico FIDO requires a **physical button press** for every operation that demands user presence. The LED pattern changes when the device is waiting for confirmation—it stays almost fully on and pulses off briefly every second.

User verification is handled by a **PIN**. The device enforces a maximum of **`MAX_PIN_RETRIES` = 8** failed attempts before the PIN is blocked. The minimum PIN length defaults to 4 and is configurable via `CTAP_CONFIG` with the `setMinPINLength` option.

## Attestation

Pico FIDO supports two attestation modes:

- **Self attestation** — the credential's own key signs the attestation statement, producing a compact response with no certificate chain.
- **Enterprise attestation** — a provisioned attestation certificate is used, enabling enterprise device tracking. Enterprise attestation can be enabled or disabled at runtime via `CTAP_CONFIG_AUT_ENABLE` / `CTAP_CONFIG_AUT_DISABLE`.
